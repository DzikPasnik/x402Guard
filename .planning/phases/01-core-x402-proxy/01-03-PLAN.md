# Plan 01-03: AppState and Shared Infrastructure

## Objective

Refactor from AppConfig as router state to a proper AppState that holds shared resources (Redis connection pool, HTTP client, config). This is required before the proxy handler can use Redis for nonces and rate limiting.

## Tasks

### Task 1: Create AppState
- Create `proxy/src/state.rs`
- Define `AppState` struct:
  ```rust
  pub struct AppState {
      pub config: AppConfig,
      pub redis: redis::Client,
      pub http_client: reqwest::Client,
  }
  ```
- Implement `Clone` (all fields are cloneable)
- Make it the Axum state type instead of `AppConfig`

### Task 2: Update main.rs initialization
- Create Redis client using `rate_limit::create_redis_client()`
- Create `reqwest::Client` with sensible defaults (timeout 30s, rustls)
- Build `AppState` and pass to router
- Add startup log with config summary (host, port, redis connected)

### Task 3: Update router.rs
- Change `Router<AppConfig>` to `Router<AppState>` throughout
- Pass `AppState` via `.with_state()`

### Task 4: Update all handlers
- Change handler state extraction from `AppConfig` to `AppState`
- Health handler: add Redis ping check (return degraded if Redis down)
- Proxy handler: extract `http_client` and `redis` from state

### Task 5: Input validation for ProxyRequest
- Validate `target_url` is a valid HTTPS URL
- Validate `target_url` is not a private/internal IP (SSRF prevention)
- Validate `x402_payment` is not empty/null
- Return 400 with clear error messages on validation failure

## Success Criteria
- AppState created and wired through entire app
- Redis connection established at startup
- Health endpoint reports Redis connectivity
- ProxyRequest validation rejects invalid inputs
