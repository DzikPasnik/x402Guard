# Plan 01-04: Proxy Handler Integration and E2E Flow

## Objective

Wire together the x402 verification, rate limiting, and nonce dedup into the actual proxy handler. Implement the full request flow: receive -> verify -> check nonce -> check rate limit -> forward -> return.

## Tasks

### Task 1: Implement full proxy handler flow
- Rewrite `forward_request()` in `proxy.rs`:
  1. Extract `AppState` from request
  2. Parse `x402_payment` as `PaymentPayload`
  3. Verify EIP-3009 signature via `x402::verify::verify_transfer_authorization()`
  4. Check nonce via `nonce::check_and_store_nonce()`
  5. Forward original request to `target_url` via `http_client`
     - Attach `X-Payment` header to forwarded request
     - Use GET/POST based on original method
  6. Return `ProxyResponse` with success status and forwarded response data

### Task 2: Wire rate limit middleware into router
- Add `RateLimitLayer` to the API router in `router.rs`
- Configure with global limit from `AppState.config.rate_limit_rps`
- Apply to `/api/v1/proxy` route specifically

### Task 3: CORS lockdown
- Replace `CorsLayer::permissive()` with configured allowed origins
- Add `ALLOWED_ORIGINS` env var (comma-separated, default: `http://localhost:3000`)
- Parse and apply in router

### Task 4: Request ID tracking
- Add `tower-http` `RequestId` layer
- Generate UUID for each request
- Include in all log spans and error responses

### Task 5: Integration tests
- Test full proxy flow: valid payment -> forwarded -> 200
- Test invalid signature -> 401
- Test replayed nonce -> 409 Conflict
- Test rate limit exceeded -> 429
- Test invalid target URL -> 400
- Test SSRF attempt (private IP) -> 400
- Use `axum::test` for in-process testing
- Mock external HTTP endpoint with `wiremock` or similar

## Success Criteria
- Valid x402 payment proxied successfully end-to-end
- Invalid signature rejected with 401
- Replayed nonce rejected with 409
- Rate limit exceeded returns 429 with Retry-After
- SSRF prevented
- Integration tests pass
