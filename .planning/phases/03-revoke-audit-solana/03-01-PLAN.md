# Plan 03-01: Immutable Audit Log System

## Goal

Capture every proxy request, guardrail violation, and session key lifecycle event in an immutable, append-only PostgreSQL table. Audit writes must be non-blocking (fire-and-forget via tokio mpsc channel) to preserve NFR-1 p99 < 200ms.

## Requirements Covered

- FR-6.1: Log every proxy request (agent, amount, target, timestamp, result)
- FR-6.2: Log all guardrail violations
- FR-6.3: Log session key creation, usage, and revocation events
- FR-6.4: Immutable append-only audit log

## New Dependencies

None — uses existing sqlx, tokio, serde_json, chrono, uuid.

## Implementation Steps

### Step 1: Database Migration (002_create_audit_log.sql)

**File:** `proxy/migrations/002_create_audit_log.sql`

Create the audit_log table with:
- `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
- `agent_id UUID REFERENCES agents(id) ON DELETE SET NULL`
- `session_key_id UUID REFERENCES session_keys(id) ON DELETE SET NULL`
- `event_type TEXT NOT NULL` with CHECK constraint for valid types
- `metadata JSONB NOT NULL DEFAULT '{}'`
- `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`

Add immutability trigger (`BEFORE UPDATE OR DELETE` → RAISE EXCEPTION).

Add indexes:
- `idx_audit_log_agent_time` on `(agent_id, created_at DESC)`
- `idx_audit_log_event_type_time` on `(event_type, created_at DESC)`
- `idx_audit_log_session_key` partial on `(session_key_id, created_at DESC) WHERE session_key_id IS NOT NULL`

### Step 2: AuditEvent Model

**New file:** `proxy/src/models/audit_event.rs`
**Modified:** `proxy/src/models/mod.rs` — add `pub mod audit_event;`

Create:
- `AuditEventType` enum (ProxyRequestReceived, ProxyRequestForwarded, ProxyRequestFailed, GuardrailViolation, SessionKeyCreated, SessionKeyUsed, SessionKeyRevoked, AllSessionKeysRevoked, AgentCreated, AgentDeactivated)
- `AuditEventType::as_str()` → `&'static str` for DB column
- `AuditEvent` struct: `agent_id: Option<Uuid>`, `session_key_id: Option<Uuid>`, `event_type: AuditEventType`, `metadata: serde_json::Value`

### Step 3: Audit Log Repository

**New file:** `proxy/src/repo/audit_log.rs`
**Modified:** `proxy/src/repo/mod.rs` — add `pub mod audit_log;`

Functions:
- `insert_event(pool, event) → Result<()>` — single INSERT
- `insert_batch(pool, events) → Result<()>` — loop of single INSERTs within background task (simple, reliable)

### Step 4: AuditWriter Background Service

**New file:** `proxy/src/services/mod.rs` — add `pub mod audit_writer;`
**New file:** `proxy/src/services/audit_writer.rs`

`AuditWriter` struct:
- Holds `mpsc::UnboundedSender<AuditEvent>`
- `spawn(pool: PgPool) -> Self` — creates channel, spawns `tokio::spawn(writer_loop)`
- `emit(&self, event: AuditEvent) -> bool` — fire-and-forget send
- `writer_loop(pool, rx)` — recv events, drain up to 64, insert batch, log errors

### Step 5: Wire AuditWriter into AppState + main.rs

**Modified:** `proxy/src/state.rs` — add `pub audit: AuditWriter` field
**Modified:** `proxy/src/main.rs` — `let audit = AuditWriter::spawn(db.clone())`, add to AppState

### Step 6: Emit Audit Events from Handlers

**Modified:** `proxy/src/handlers/proxy.rs` — emit at:
1. After input validation: `ProxyRequestReceived`
2. After guardrail violation: `GuardrailViolation` (with rule details in metadata)
3. After successful forward: `ProxyRequestForwarded` (with upstream status)
4. After forward failure: `ProxyRequestFailed`

**Modified:** `proxy/src/handlers/session_keys.rs` — emit at:
1. After create: `SessionKeyCreated` (public_key, max_spend, expires_at)
2. After revoke: `SessionKeyRevoked`

**Modified:** `proxy/src/handlers/agents.rs` — emit at:
1. After create: `AgentCreated` (name, owner_address)

### Step 7: Unit Tests

Tests (no DB required — mock AuditWriter):
- AuditEventType::as_str() returns correct strings for all variants
- AuditEvent serialization to JSON is correct
- AuditWriter::emit returns true when channel open
- AuditWriter::emit returns false when channel closed (writer crashed)

Integration test (requires Docker+Postgres):
- Insert event via repo, verify it exists in DB
- Verify UPDATE on audit_log raises exception (immutability trigger)
- Verify DELETE on audit_log raises exception
