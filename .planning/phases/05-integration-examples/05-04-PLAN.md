3. **tsup.config.ts**: entry src/index.ts, format esm, dts true, clean true, sourcemap true, external `@x402guard/core`, `@cod3x/sdk`, `@cod3x/ethers`.

4. **src/types.ts**: Cod3x-specific type extensions:
   - `Cod3xPaymentRequest`: extends ProxyRequest with optional `cod3xSessionId?: string` for Cod3x session tracking
   - `Cod3xGuardedResponse`: extends ProxyResponse with `guardedBy: 'x402guard'` marker field
   - `X402GuardCod3xAdapterConfig`: `{ proxyUrl?: string, agentId?: string, logLevel?: string, maxRetries?: number }` -- mirrors X402GuardConfig but adds Cod3x-specific docs
   - `Cod3xSecurityWorkerConfig`: Type for Cod3x security worker integration `{ securityWorker: (data: unknown) => Promise<{ headers: Record<string, string> }> }`

5. **src/adapter.ts**: Implement `X402GuardCod3xAdapter`:
   - Constructor takes `X402GuardCod3xAdapterConfig`, creates internal `X402GuardClient` from `@x402guard/core`
   - Config frozen after construction (immutability rule)
   - Methods (all async):
     - `healthCheck(): Promise<boolean>` -- delegates to X402GuardClient.healthCheck()
     - `guardedExecute(request: Cod3xPaymentRequest): Promise<Cod3xGuardedResponse>` -- calls X402GuardClient.proxyPayment() and wraps response with `guardedBy: 'x402guard'`. Catches and re-throws GuardrailViolationError from core.
     - `guardedSolanaExecute(request: SolanaProxyRequest): Promise<SolanaProxyResponse>` -- calls X402GuardClient.proxySolanaPayment()
     - `createAgent(name: string, ownerAddress: string): Promise<Agent>` -- convenience delegate
     - `createGuardrail(agentId: string, ruleType: RuleType): Promise<GuardrailRule>` -- convenience delegate wrapping createRule
     - `revokeAllKeys(agentId: string, ownerAddress: string): Promise<void>` -- convenience delegate
     - `toSecurityWorker(): Cod3xSecurityWorkerConfig` -- returns a Cod3x-compatible security worker that injects x402Guard agent ID into request headers. Pattern: `{ securityWorker: async () => ({ headers: { 'X-Agent-Id': this.agentId } }) }`. This is the integration point for @cod3x/sdk users.
   - JSDoc on all public methods with usage examples
   - No console.log (use createLogger from @x402guard/core)

6. **src/index.ts**: Re-export adapter, types, and all @x402guard/core exports.

7. **tests/unit/adapter.test.ts** (vitest):
   - Mock `global.fetch` with `vi.spyOn(global, 'fetch')`
   - Test: adapter.healthCheck delegates to client and returns boolean
   - Test: adapter.guardedExecute returns Cod3xGuardedResponse with guardedBy field on success
   - Test: adapter.guardedExecute throws GuardrailViolationError on 403
   - Test: adapter.createGuardrail delegates to client.createRule
   - Test: adapter.toSecurityWorker returns object with securityWorker function
   - Test: securityWorker function returns headers with X-Agent-Id
   - Test: constructor throws on missing proxyUrl

   - Step 3: Set guardrail via adapter.createGuardrail with MaxSpendPerTx limit 1_000_000
   - Step 4: Make guarded payment via adapter.guardedExecute with amount under limit. Use `viem` to construct a valid EIP-712 TransferWithAuthorization signature with random nonce and deterministic test private key (0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80 -- Hardhat #0, labeled TEST ONLY).

     ```
       limit:     1,000,000 (1.00 USDC)

     ```
   - Step 6: Demonstrate toSecurityWorker() integration point -- show the security worker object structure
   - All config from env vars. SECURITY: test key labeled TEST ONLY.
