Scripts: `build`, `test`, `test:integration`, `demo` (tsx demo/demo.ts).

2. **tsconfig.json**: Same base config as core package. Add path alias for `@x402guard/core`.

3. **tsup.config.ts**: entry src/index.ts, format esm, dts true, clean true, sourcemap true, external `@elizaos/core` and `@x402guard/core`.

4. **src/actions/guardedPayment.ts**: Implement `guardedPaymentAction` as an ElizaOS Action:
   - `name`: 'GUARDED_PAYMENT'
   - `description`: 'Make an x402 payment guarded by x402Guard guardrails'
   - `similes`: ['PAY', 'TRANSFER', 'SEND_PAYMENT', 'X402_PAY']
   - `validate`: Returns true if `runtime.getSetting('X402GUARD_PROXY_URL')` is truthy
   - `handler`: Creates X402GuardClient from runtime settings, extracts payment params from message content (targetUrl, amount, network), calls `proxyPayment` or `proxySolanaPayment` based on network. On success: callback with success text. On GuardrailViolationError: callback with structured violation details (ruleType, limit, actual). On other errors: callback with error message.
   - Handler must extract `X402GUARD_PROXY_URL` and `X402GUARD_AGENT_ID` from `runtime.getSetting()`.
   - IMPORTANT: The handler does NOT construct EIP-712 signatures. The agent runtime provides pre-signed payment payloads. The handler wraps them in ProxyRequest and sends to the proxy.
   - Include JSDoc on all exported symbols.

5. **src/providers/x402guard.ts**: Implement `x402guardProvider` as an ElizaOS Provider:
   - `name`: 'X402GUARD_STATUS'
   - `description`: 'Provides x402Guard proxy connection status and agent info to context'
   - `get`: Creates X402GuardClient, calls `healthCheck()`. Returns `{ text: "x402Guard proxy is reachable at {url}", data: { healthy: true, proxyUrl } }` on success, `{ text: "x402Guard proxy unreachable at {url}", data: { healthy: false } }` on failure.

6. **src/plugin.ts**: Export `x402guardPlugin` implementing Plugin interface:
   - `name`: '@elizaos/plugin-x402guard'
   - `description`: 'x402Guard guardrail proxy integration for ElizaOS agents'
   - `config`: `{ X402GUARD_PROXY_URL, X402GUARD_AGENT_ID }` with env var fallbacks
   - `init`: Create client, call healthCheck(), throw with actionable message if proxy unreachable: "x402Guard proxy unreachable at {url} -- is docker compose up running?"
   - `actions`: [guardedPaymentAction]
   - `providers`: [x402guardProvider]

7. **src/index.ts**: Re-export plugin, action, provider, and all types from @x402guard/core.

8. **tests/unit/plugin.test.ts** (vitest):
   - Test: plugin has correct name and description
   - Test: plugin.actions contains guardedPaymentAction
   - Test: plugin.providers contains x402guardProvider
   - Test: plugin.config has expected keys

9. **tests/unit/actions.test.ts** (vitest):
   - Mock `global.fetch` and create mock IAgentRuntime with `getSetting` returning test values
   - Test: guardedPaymentAction.validate returns true when X402GUARD_PROXY_URL is set
   - Test: guardedPaymentAction.validate returns false when X402GUARD_PROXY_URL is missing
   - Test: guardedPaymentAction.handler calls proxyPayment and returns success
   - Test: guardedPaymentAction.handler catches GuardrailViolationError and reports violation details

No console.log anywhere in src/ (project rule). Use pino logger from @x402guard/core.
