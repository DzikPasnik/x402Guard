Cargo workspace structure — critical for Docker build:
- Workspace root: /app (in container)
- proxy/Dockerfile WORKDIR is /app
- proxy/Dockerfile copies Cargo.toml and Cargo.lock from workspace root (COPY Cargo.toml Cargo.lock* ./)
- proxy/Dockerfile copies proxy/Cargo.toml into proxy/Cargo.toml
- proxy/Dockerfile copies proxy/src into proxy/src
- Therefore: docker-compose build context MUST be the workspace root (.), not proxy/

Services needed:
- proxy: Rust binary on port 3402
- postgres: PostgreSQL 16 on 5432 (Supabase-compatible)
- redis: Redis 7 on 6379

Environment variables proxy needs (from config.rs):
- PROXY_HOST (default 0.0.0.0)
- PROXY_PORT (default 3402)
- BASE_SEPOLIA_RPC_URL
- BASE_MAINNET_RPC_URL
- DATABASE_URL -> must use compose service name "postgres"
- UPSTASH_REDIS_URL -> must use compose service name "redis"
- RUST_LOG

<task type="auto">
  <name>Task 1: Create docker-compose.yml</name>
  <files>docker-compose.yml</files>
  <action>
Create docker-compose.yml at the workspace root (D:/x402Guard/docker-compose.yml) with the following:

```yaml
version: "3.9"

services:
  proxy:
    build:
      context: .
      dockerfile: proxy/Dockerfile
    ports:
      - "3402:3402"
    environment:
      PROXY_HOST: "0.0.0.0"
      PROXY_PORT: "3402"
      RUST_LOG: "info,x402_guard_proxy=debug"
      DATABASE_URL: "postgresql://postgres:postgres@postgres:5432/postgres"
      UPSTASH_REDIS_URL: "redis://redis:6379"
      BASE_SEPOLIA_RPC_URL: "https://sepolia.base.org"
      BASE_MAINNET_RPC_URL: "https://mainnet.base.org"
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped

  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: postgres
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5
    command: redis-server --appendonly yes

volumes:
  postgres_data:
  redis_data:
```

Key requirements:
- Build context is `.` (workspace root) so the Dockerfile can copy Cargo.toml and Cargo.lock from root
- Dockerfile path is `proxy/Dockerfile` (relative to workspace root context)
- DATABASE_URL uses hostname "postgres" (compose service name, not localhost)
- UPSTASH_REDIS_URL uses hostname "redis" (compose service name, not localhost)
- Both postgres and redis have healthchecks so proxy waits for them to be ready
- Volumes are named to persist data between restarts
- proxy depends_on with condition: service_healthy for both deps
