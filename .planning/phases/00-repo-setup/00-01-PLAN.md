# Plan 00-01: Next.js Dashboard Skeleton

## Objective

Create the Next.js 15 dashboard skeleton with App Router, Tailwind CSS, and shadcn/ui. The dashboard will be the user's primary interface for monitoring agents, configuring guardrails, and managing session keys.

## Context

Previous attempt to run `create-next-app` interactively failed. Must use fully non-interactive flags.

## Tasks

### Task 1: Create Next.js 15 project
- Run `npx create-next-app@latest dashboard` with all flags to avoid interactive prompts:
  `--typescript --tailwind --eslint --app --src-dir --import-alias "@/*" --use-npm --no-turbopack`
- If the `dashboard/` directory already exists from a failed attempt, remove it first
- Use `yes |` or `echo "no" |` pipe to handle any remaining prompts

### Task 2: Install shadcn/ui
- Run `npx shadcn@latest init` in dashboard directory with `--defaults` flag
- Install base components: `button`, `card`, `input`, `badge`, `table`, `alert`

### Task 3: Install Supabase client
- `npm install @supabase/supabase-js`
- Create `dashboard/src/lib/supabase.ts` with client initialization from env vars

### Task 4: Create dashboard landing page
- Replace `dashboard/src/app/page.tsx` with a simple landing page showing:
  - x402Guard logo/title
  - "Dashboard coming in Phase 4" placeholder
  - Link to health endpoint
- Create `dashboard/src/app/layout.tsx` with proper metadata

### Task 5: Verify dashboard starts
- Run `npm run build` to verify build succeeds
- Run `npm run dev` briefly to verify dev server starts

## Files Modified
- `dashboard/` (entire directory created)
- `dashboard/src/lib/supabase.ts`
- `dashboard/src/app/page.tsx`
- `dashboard/src/app/layout.tsx`

## Success Criteria
- `npm run build` passes without errors
- `npm run dev` starts and serves landing page
- shadcn/ui components importable
